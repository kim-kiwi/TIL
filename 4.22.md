## 인증과 인가
### 인증
대상이 '누구인지' 확인하는 신원 검증 과정
### 인가
어떤 대상이 어떤 자원에 접근할 수 있는지 판단하는 과정

## Spring Security
인증, 권한 관리 그리고 데이터 보호 등등 보안 관련 이것저것 해주는 프레임워크

### 기본 용어
 - 접근 주체(Principal): 보호된 자원에 접근하는 사용자 또는 시스템 
 - 인증(Authentication): [인증과 인가](#인증과-인가) 
 - 인가(Authorization): [인증과 인가](#인증과-인가)
 - 권한: 어떤 대상이 특정 행위가 가능한지를 결정할 때 사용

### 특징
 - 보안과 관련하여 체계적으로 많은 기능을 제공한다.
 - Filter를 기반으로 동작하여 MVC(Model-View-Controller)와 구분하여 관리 및 동작한다.
 - 기본적으로 세션 & 쿠키 방식으로 인증한다.

## JWT
**J**son **W**eb **T**oken

서버랑 클라이언트가 json을 이용하여 가볍고 안전하게 정보를 전달하는 방법.
### 특징
 - Stateless하다(session은 Stateful)
 - 토큰 자체에 필요한 정보를 인코딩해 저장한다.
 - 비밀키를 통해 위조 여부를 감지한다.
 - DB에 무언갈 저장하거나 하지 않아 자원 절약이 된다.

### 구조
 - Header: 토큰 타입 및 서명 알고리즘 정보를 담고있다
 - Payload: base64로 인코딩된 실제 정보(Claim이라고 함)를 담고있다
 - Signature: 헤더와 페이로드를 비밀키로 서명하여 위조를 감지한다.

### 용도
 - 사용자 인증
 - 분산 서비스에서 하나의 토큰으로 여러 서비스 접근 권한 부여
 - 정보 교환 시 데이터 무결성 보장
 - 무상태 서버를 만들고 싶을때 사용

## Spring Bean
스프링 컨테이너(IoC 관리하는 놈)가 생성, 의존성 설정, 생명주기 관리 등을 수행하는 자바 객체이다.

### 주요 개념
 - 스프링 컨테이너: 빈의 생성, 의존성 주입, 생명주기 관리 등을 수행. IoC와 DI를 제공
   - 예: BeanFactory, ApplicationContext
 - 빈(Bean): 스프링 컨테이너에서 관리되는 인스턴스화된 자바 객체

### 특징
 - 기본적으론 스프링 컨테이너당 하나의 빈 인스턴스만 생성한다.
 - 빈의 생명주기를 스프링 컨테이너가 관리한다.
 - 어노테이션을 통해 컨테이너에 등록된다.
