# RESTful api와 DB

## RESTful API
 - http 표준 메서드를 사용한다.
 - API에서 다루는 리소스는 URL로 표현된다.
 - 상태가 없다.
 - json, xml을 사용한다.

예시: `GET /users/123` (123 유저를 조회)

## DB
 - 데이터를 저장하는 역할을 한다.

개념: 
 - DBMS(Database Management System): 데이터베이스 생성 관리 소프트웨어. 예: MySQL
 - 테이블: DB의 데이터를 나타내는 기본 단위이다. 행과 열로 구성되어 있다.
 - 레코드: 테이블의 한 행이다. 하나의 데이터 단위를 말한다. (고객 테이블에서 한 레코드는 고객 한명에 해당한다)
 - 필드: 테이블의 열을 말하며 하나의 속성을 의미한다. (고객 테이블에서 한 필드는 각 고객의 특정 정보가 된다. 예를 들어 나이, 이름, 주소 등등)
 - 쿼리: 데이터를 검색, 요청, 조작하기 위해 DB에 보내는 명령어다.

종류:
 - 관계형 DB
   - 표 형식이다.
   - SQL를 사용한다.
   - 예: MySQL
 - 비관계형 DB
   - 표 형식이 아닌 다양한 형태를 사용한다. (키-값 쌍, 문서, 그래프 등등)
   - 예: MongoDB
